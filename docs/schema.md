# Unified Review JSON Schema

This schema is used across all pipeline stages for the Blinkit Discovery Engine to standardize data from various sources (Reddit, App Store, Play Store, etc.).

```json
{
  "id": "uuid-v4",
  "source": "Reddit | Play Store | App Store | Twitter | Forum",
  "platform": "Blinkit | Instamart | Zepto",
  "text": "Raw review text (PII stripped)",
  "rating": 4,
  "timestamp": "2023-10-01T12:00:00Z",
  "url": "https://...",
  "enrichment": {
    "themes": ["Trust - Fresh Produce", "Habit Loop - Reordering"],
    "sentiment": "Negative",
    "segment": ["Pet Owner", "Grocery Buyer"],
    "unmet_needs": ["Better category discovery", "Return policy clarity"]
  }
}
```

### Field Descriptions

- **`id`**: Unique identifier for the review (UUID v4).
- **`source`**: The platform where the review was collected from.
- **`platform`**: The target quick-commerce application the review is about.
- **`text`**: The actual content of the review (ensure all PII like emails/phone numbers are stripped before saving).
- **`rating`**: Numeric rating (1-5) if applicable. Can be `null` for sources like Reddit or Forums.
- **`timestamp`**: ISO 8601 formatted timestamp of when the review was posted.
- **`url`**: A direct link to the original review/comment.
- **`enrichment`**: Metadata generated by the Groq API LLM processing.
  - **`themes`**: Identified recurring reasons for shopping behavior or frustrations.
  - **`sentiment`**: Categorized as `Positive`, `Neutral`, or `Negative`.
  - **`segment`**: Discovered user segments/personas (freely inferred from text).
  - **`unmet_needs`**: Explicit gaps or feature requests mentioned by the user.
